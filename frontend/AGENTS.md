# frontend/AGENTS.md

React 18 zero-build frontend: vanilla JS modules, no transpilation needed for production.

---

## OVERVIEW

Single-page app using React 18 UMD + Babel standalone. Fetches `data/data.json` (written by backend), detects region from IP, displays gas price + timing advice.

---

## STRUCTURE

```
frontend/
├── index.html         Entry point: React 18 + Babel standalone CDN imports
├── css/styles.css     Global styles (Hanken Grotesk + Bricolage Grotesque fonts)
├── data/data.json     Auto-generated by backend (Approach A - no API server)
└── js/
    ├── data.js        Data model + detectRegionFromIP() (all 62 regions)
    ├── components.js  Presentational components (LocationSheet, GasPriceDisplay, etc.)
    └── app.js         Root App component + WTI state management
```

## CONVENTIONS

### Zero-build architecture
- **No build step**: Serve `frontend/` with any static server (`python -m http.server`)
- React 18 UMD + Babel standalone loaded from CDN (unpkg.com with SRI integrity hashes)
- ES modules (`type="text/babel"`) transpiled in-browser by Babel standalone
- Production-ready: Works on `file://` for offline use (IP geolocation degrades gracefully)

### Unit display
- **US regions**: Show `$/gal` (e.g., `$3.45/gal`)
- **CA regions**: Show `$/L` (e.g., `$1.52/L`)
- Check `region.unit` field from data.json - NEVER hardcode unit strings

### Region detection
- On load: `detectRegionFromIP()` silently maps IP → state/province (no permission prompt)
- Falls back to Ontario (`on`) for unmatched Canadian IPs, California (`ca`) for unmatched US IPs
- "Use My Location" button → browser geolocation API (opt-in) for precise coordinates

### Data shape
Each region in `data.json`:
```js
{
  id: "ca",
  state: "California",
  abbr: "CA",
  city: "Los Angeles",
  country: "US",
  unit: "gal",
  verdict: "buy" | "wait" | "partial",
  price: 3.45,
  priceLow: 3.21,
  weekDelta: 0.12,
  why: "Crude oil dropped 3% this week...",
  advice: "Fill up today",
  bestDayIdx: 2,
  wtiDir: "down" | "up" | "flat",
  breakdown: { crude: 61, refining: 14, taxes: 14, dist: 11 },
  trend: [3.21, 3.34, 3.45, ...],  // last 14 days
  news: [{ headline, source, url }, ...]
}
```

## ANTI-PATTERNS

- **Introducing a build step**: This is a **zero-build project** - do NOT add Webpack/Vite/Parcel unless explicitly changing architecture
- **Hardcoding units**: ALWAYS use `region.unit` from data - NEVER assume $/gal
- **Breaking data.json contract**: Frontend expects `data/data.json` at load - do NOT change path without updating backend `DATA_JSON_PATH`
- **Over-engineering state**: Keep state minimal (selected region, WTI) - most data flows from data.json

## NOTES

- **React 18 + Babel standalone**: UMD builds (react.development.js, react-dom.development.js) + babel.min.js from CDN
- **SRI integrity hashes**: Locked to specific versions (React 18.3.1, Babel 7.29.0) for security
- **Fonts**: Hanken Grotesk (body) + Bricolage Grotesque (headings) from Google Fonts
- **IP geolocation**: Uses ipapi.co free tier (silent, no signup) - falls back gracefully if unavailable
- **GasBuddy deep-link**: For exact station prices, we redirect to GasBuddy (we don't duplicate their core feature)
